# Calculator 2.0 Compatibility Matrix

This file is the Phase 0 baseline required by `VERSION_2_IMPLEMENTATION_PLAN.md`.
It records the 1.0 public surface that 2.0 must preserve while the new runtime is
built alongside the legacy implementation.

| Area | 1.0 status | 2.0 migration target |
| --- | --- | --- |
| Scalar arithmetic | Supported by `DecimalParser` and exact side path | Route through `numeric::Number` and unified expressions |
| Scientific functions | Supported by `mymath` double backend | Reimplement precision-aware numeric and complex handlers |
| Exact rationals | Supported with `long long` limits | Replace with `numeric::Rational` over `numeric::BigInt` |
| Precise decimals | Supported as a separate string-decimal path | Replace with `numeric::BigDecimal` and `PrecisionContext` |
| Complex values | Limited helper coverage | Promote to first-class `NumberKind::Complex` |
| Matrices | `double` matrix fast path | Add `MatrixValue` over runtime `Value` while retaining fast paths |
| Symbolic expressions | Side subsystem with `double` numeric nodes | Replace numeric nodes with exact 2.0 numbers |
| Differentiation | Existing symbolic rules | Migrate to canonical CAS expression tree |
| Integration | Existing staged symbolic/numeric support | Return closed forms where supported, otherwise unevaluated `Integral` |
| Variables/functions | `StoredValue` plus custom-function maps | Typed runtime `Environment` and function registry |
| Scripts | Existing AST/runtime | Migrate bindings and values to 2.0 runtime |
| Persistence | 1.0 state text format | Load 1.0 files and write versioned 2.0 state |

Initial 2.0 implementation status:

- Added `src/numeric` with `BigInt`, `Rational`, `BigDecimal`, `Complex`,
  `Number`, and `PrecisionContext`.
- Added unit coverage for large integer arithmetic, exact rationals,
  precision-controlled decimal conversion, promotion rules, and `i * i = -1`.
- Added `src/expression` and `src/runtime` with a unified expression tree,
  parser, evaluator, printer, value model, environment, and line executor.
- Added `src/runtime/function_registry.*` with names, aliases, categories, help
  text, autocomplete examples, derivative/integration metadata, arity metadata,
  and exact-first evaluators for the migrated Phase 3 functions.
- Added a compatibility bridge exposed through `:v2`, `:v2vars`, `:v2clear`,
  and `:v2precision`; complex literals using `i` are routed to the 2.0 runtime.
- Added `src/cas` with canonical local simplification and symbolic
  differentiation for the Phase 4 expression forms currently represented by the
  new expression tree.
- Added `src/cas/integrate.*` with a layered symbolic integration pass and a
  first-class unevaluated `Integral` expression node for unsupported forms.
- Exported the 1.0-to-2.0 function checklist in
  `docs/2.0_FUNCTION_COVERAGE.md`.

Phase 3 completed in the new 2.0 core scope:

- constants: `i` is numeric; `pi` and `e` are preserved symbolically until
  high-precision real constants are implemented
- exact arithmetic and promotion through `Number`
- function registry metadata and alias lookup
- migrated exact-first functions: `sqrt`, `pow`, `real`, `imag`, `conj`, `abs`,
  `sign`, `step`, `heaviside`, `delta`, `impulse`, `floor`, `ceil`, `trunc`,
  `round`, `min`, `max`, `clamp`, `sum`, `mean`, `avg`, `median`, `mode`,
  `var`, `variance`, `idiv`, `mod`, `gcd`, `lcm`, `factorial`, `nCr`,
  `binom`, `nPr`, `fib`, `fibonacci`, `is_prime`, `next_prime`, `prev_prime`,
  `prime_pi`, `euler_phi`, `phi`, `celsius`, `fahrenheit`, `kelvin`, `c2f`,
  and `f2c`
- exact known-value handlers for `sin`, `cos`, `tan`, `exp`, `ln`, `log`,
  `sinh`, `cosh`, `tanh`, `asin`, `atan`, `asinh`, and `atanh`
- unsupported high-precision transcendental numeric cases intentionally remain
  symbolic instead of falling back to the 1.0 double backend

Phase 4 completed in the new 2.0 core scope:

- simplification: identity removal, exact constant folding, repeated-factor
  powers, inverse powers, same-base power merging, `sin(x)^2 + cos(x)^2`, and
  `exp(ln(x))` / `ln(exp(x))`
- differentiation: constants, variables, addition, product rule, power rule
  including `x^x`, chain rule, trigonometric, inverse trigonometric,
  hyperbolic, logarithmic, exponential, `sqrt`, and unevaluated `Derivative`
  output for unknown functions
- multi-variable CAS helpers: `gradient`, `jacobian`, and `hessian`

Phase 5 completed in the new 2.0 core scope:

- first-class unevaluated `Integral` expression output
- constant, linearity, power, polynomial, `1/x`, logarithm, trigonometric,
  reciprocal trigonometric, exponential, and linear-argument table rules
- simple substitution detection including `u' * f(u)` and `u'/u`
- selected rational forms including `1/(a*x+b)`, `1/(x^2+1)`, and
  `(2*x)/(x^2+1)`
- recursive integration by parts for `x^n*exp(x)`, `x^n*sin(x)`, and
  `x^n*cos(x)`
- special-function outputs for `exp(-x^2)` and `exp(x^2)` through `erf` and
  `erfi`
