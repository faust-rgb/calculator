# Problem List Resolution Notes

本文记录 `problem.txt` 中问题与扩展方向的处理结果。

## 已修改

- 符号三角积分扩展
  - 新增 `sec(a*x+b)`、`csc(a*x+b)`、`cot(a*x+b)` 的不定积分规则。
  - 新增 `sec(x)^2`、`csc(x)^2`、`cot(x)^2` 的幂积分规则。
  - 新增 `sec(x)^2 * tan(x)` 与 `csc(x)^2 * cot(x)` 的常见乘积积分规则。
  - 新增 `sec`、`csc`、`cot` 的符号求导规则，保证新增积分表达式能继续参与符号运算。

- 数值积分增强
  - 将一元定积分主路径从自适应 Simpson 换为自适应 Gauss-Kronrod G7-K15。
  - G7-K15 内置误差估计，且不采样端点，对端点奇异函数更稳。
  - 对端点不可求值或非有限值的定积分增加变量替换，覆盖 `integral(1/sqrt(x), 0, 1)` 这类端点奇异积分。
  - 增加高频振荡函数回归测试，改善纯 Simpson 路径对振荡函数收敛慢的问题。

- 数值微分增强
  - 一阶数值导数改为自适应步长中心差分加 4 层 Richardson 外推。
  - 增加 `exp(x)` 非零点导数回归测试，覆盖更高阶外推路径。

## 暂未修改及原因

- Risch 算法、完整换元积分框架、完整 LIATE 分部积分库
  - 这些属于完整 CAS 级符号积分工程，需要表达式域、条件域、函数域扩展；当前只补充局部规则，避免引入不完整的大框架。

- 完整部分分式分解的任意多个不同不可约二次因子
  - 现有代码已支持实一次因子和一个不可约二次因子族。多个不同二次因子需要更通用的因式分解和稳定线性系统构造，暂保留为后续中优先级工作。

- L'Hopital 符号极限
  - 当前极限命令仍以数值采样为主。加入 L'Hopital 需要先把命令层极限与符号表达式域更紧密地打通，避免在方向极限、绝对值、不可导点上给出错误结论。

- 无穷区间积分
  - 解析器和命令层目前没有稳定的 `inf`/`infinity` 字面量协议。本次只处理有限区间端点奇异积分；无穷区间应和输入语法一起设计。

- 隐函数微分、参数曲线微分、方向导数、曲线积分、面积分
  - 多变量符号微分已有基础梯度/Jacobian/Hessian 能力，但这些新命令需要命令语法、输出格式和文档示例同步设计，未在本轮混入。

- 复数矩阵、复数多项式求根、复数特殊函数、复数字面量语法糖
  - 会影响解析器、矩阵值类型、函数分派和显示层。当前数值核心仍以实数 `double` 为主，不适合局部打补丁。

- FFT 混合基、Bluestein、实数 FFT、FFT 卷积、循环卷积、相关函数、窗函数扩展、滤波器设计、PSD/STFT/spectrogram
  - 这些属于信号处理子系统建设，现有代码没有独立 DSP 模块。本轮未创建新子系统，只记录为后续模块化扩展方向。

- 任意精度超越函数、可配置精度、高精度矩阵
  - 当前高精度能力主要在输入/显示和部分内部路径，核心数学函数和矩阵算法仍以 `double` 为基础。完整高精度需要统一数值类型抽象。
